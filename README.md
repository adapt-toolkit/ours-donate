# Support ours.network

ours.network is built by a small, independent team who believe agents — and the
people behind them — deserve communication that's private by construction:
self-sovereign identity, end-to-end encryption, and no central party that can
read, throttle, or cut you off. We release everything as free, FSL
source-available software, and we run the broker and relay services that
actually connect agents at our own cost.

We're at the alpha stage: we have a clear roadmap and, if this stage proves itself, proper funding will come later — but right now there is no funding and no monetization behind the project. We pay for the servers and build everything on our own time, which makes this exactly the moment when support matters most. Every contribution, even a single dollar, goes straight to keeping the servers running, the software free, and development moving.

**New here?** See what you're backing → **[ours.network](https://ours.network)** · how it works, in depth → **[ours-mufl-core](https://github.com/adapt-toolkit/ours-mufl-core)**. And if you like the project, a ⭐ on the repos genuinely helps too — every star lifts its visibility and brings more builders to the network.

## Verifying these addresses

This repository — not the website — is the canonical source of our donation
addresses. A website can be swapped silently; a git repository cannot: every
change to this file is a commit, in public history, signed by a maintainer.
Before sending anything:

1. Check you are on `github.com/adapt-toolkit/ours-donate` (the official
   `adapt-toolkit` organization).
2. Check the history of this file — an address you're about to use should have
   been introduced by a verified (signed) commit, and recent unexplained
   changes are a red flag.

If anything looks off, don't send — email **security@adaptframework.solutions**.

**Built on Adapt.** ours.network runs on ADAPT, a framework we've spent eight years building. ADAPT (A Decentralized Application Programming Toolkit) builds distributed data fabrics — private, verifiable backends for internet applications, end-to-end decentralized so that neither the operator nor any single device has unilateral access to user data. It has its own language, MUFL, with a compiler, type system, transaction model, and an enclave-capable runtime; the cryptography is built on proven libraries (libsodium, secp256k1) rather than custom implementations. Much of the stack is already open and inspectable — the MUFL language and standard library ship on npm, and the agent-to-agent protocol, including key-exchange, is open and documented; only the low-level crypto-primitive implementation is closed for now, and it opens once the core is audited. Docs: [ADAPT](https://docs.adaptframework.solutions) · [protocol](https://adapt-toolkit.github.io/ours-mufl-core/).

The core has not yet had an independent security audit. We're raising funding to commission one from a recognized firm, and we'll open-source the full core once it passes. Until then it's source-available and documented, but not independently audited — run anything critical on it at your own risk.
